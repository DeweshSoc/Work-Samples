Unix Command Line


If the operating system is a pistachio (pista) then the actual inner nut is the kernel.
In computing -

KERNEL - program that interfaces directly with the hardware. So, it's the core of the operating system.
SHEEL - user interface for us as humans to be able to interact with the kernel and in turn with the hardware of the computer.

There are two variants to the shell:
GUI - Graphical User Interface shells. Finder in MacOS, Explorer in Windows 10 are both GUIs.
CLI - Command Line Interface shells.

BASH SHELL: Bourne Again Shell
The Bash Shell is a CLI, command line interpreter for the UNIX system. Now, we are learning this because there are many UNIX based systems or UNIX like systems. Eg: LINUX is a UNIX like system, many servers run on UNIX, MacOS runs on a UNIX like system.
Now, this is a completely different family from the Windows, which uses DOS.

Advantages of using Command Line:

  • greater control and speed
  • greater flexibility


We will use HYPER terminal: link to hyper

Command line Techniques and Directory Navigation:

List Command : [syntax- ls]
It lists all the items inside a particular directory, including any items that were marked hidden.
In HYPER, ~ (tilde) signifies that we are accessing the user directory of the particular user who has opened HYPER.

Change Directory : [syntax- cd[address]]
BY default HYPER or CMD takes us to the location ~, which is the user directory. But if we want to navigate to some other directory then we use change directory command.

If a directory name has more than one word (spaces in between), then to signify space we use backslash (\) and to navigate back to the parent directories we use two dots (..) after cd.

Root directory of the entire system is accessed by using forward slash (/) after cd. Root directory is the highest level directory of the hierarchy of the entire computer, and in BASH if we want to change drive, we need to navigate to the root directory first.It is like a master directory that is accessed by a forward slash and contains all the drives and also some important programs and other stuff.

File / Directory - Creation and Manipulation:

  • To create a new directory we use the make directory command syntax- mkdir[name]
  • To create a file we use touch command syntax- touch[name.extension]
  • To open a file we use the start command syntax- start[name.extension]
  • To open a file in a particular application we use, syntax- start[app name] [name.extension]
  • To delete a file we use remove command rm[file_name.extension]
  • To delete all files in a directory we use, synatx- rm* To delete a directory we use, syntax- rm -r [directory name]

The last three are destructive commands If we use these commands on imporatnt files/folders then these will remove all the very important data / stuff that runs the computer. All the more important when running servers because one remove all command on a server will erase data for many users. Therefore, these commands should be used only when absolutely necessary and always keep a check of the directory you are working in before executing these commands.
